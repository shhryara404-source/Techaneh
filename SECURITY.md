# سیاست امنیتی تکانه

## گزارش آسیب‌پذیری

آسیب‌پذیری‌ها را به‌صورت عمومی در issue منتشر نکنید. یک مسیر تماس خصوصی و کنترل‌شده برای مالک پروژه تعریف شود.

## قواعد کلیدی

- هیچ API Key در Git، فایل‌های JavaScript سمت کاربر، عکس، PDF عمومی یا پیام‌رسان قرار نگیرد.
- کلید مقاله و کلید دستیار جدا نگه داشته شوند.
- Secretهای GitHub فقط در GitHub Actions Secrets ذخیره شوند.
- Secretهای دستیار فقط در Cloudflare Worker Secrets ذخیره شوند.
- دامنه‌های مجاز Worker محدود به `techaneh.ir`، `www.techaneh.ir` و آدرس موقت GitHub Pages باشند.
- Turnstile و Rate Limit قبل از تبلیغ عمومی دستیار فعال شوند.
- وابستگی‌ها و Actionها به نسخه‌های معتبر و رسمی محدود شوند.
- پاسخ عامل و داده وب قبل از انتشار عمومی به‌عنوان داده غیرقابل‌اعتماد اعتبارسنجی شوند.

## چرخش کلید

در صورت نمایش ناخواسته یک کلید:

1. همان لحظه کلید را در سرویس صادرکننده لغو کنید.
2. کلید جدید بسازید.
3. Secret مربوط را جایگزین کنید.
4. تاریخچه commit و logها را بررسی کنید.
5. مصرف و هزینه غیرعادی را کنترل کنید.
